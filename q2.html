  2. CSS Selectors
    • states which element in the HTML you want to style/part of the statement which identifies page elements
    • element selectors: selects HTML elements (p, div, h1, etc) and applies CSS to them
    • id selectors: selects the HTML element with a unique identifier (id) and adds CSS to it
    • class selectors: selects the HTML element using the class attribute and applies CSS to it
    • universal selectors: selects every single HTML element on the page. It is written using the asterisk ( * ) character
    • group selectors: allows you to select multiple elements and apply the same style to all of them.

Reflection

During the second quarter, I learned a lot about Cascading Style Sheets (CSS) and how it helps make websites look better. CSS is a language we use to style HTML elements, which means it tells the website how things should look. By using external stylesheets, I can keep the design separate from the content. This makes it easier to update and change the look of a page without affecting the text or pictures. I also learned that there are three main parts to a webpage: the content (HTML), the presentation (CSS), and the behavior (JavaScript).

I explored different parts of CSS, like selectors and properties that help control how elements are shown on a webpage. I now understand the differences between inline styles, embedded styles, and linked styles. Each one has its own benefits for styling. I learned how to use selectors to choose specific parts of an HTML page to style, like using element selectors, class selectors, and ID selectors. Understanding how styles "cascade" helps me fix problems when different styles affect the same elements.

Finally, we talked about how to use the box model and control the layout of the webpage. I learned about properties for displaying elements and how to position them exactly where I want. We also covered pseudo-classes, which allow me to change the style of an element when someone interacts with it, like when they hover over a link. ​Overall, this quarter was really helpful, and I feel like I have a strong understanding of CSS that I can use for web development in the future.
